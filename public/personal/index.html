The Remote Terminal Emulator emulates a BASH terminal session on a remote server. This project has a client server architecture, including networking sockets, a thread pool dealing with integers, and UNIX timers to handle malicious clients. All I/O operations are handled by an epoll loop, which waits for commands from the client, then relays the client's commands to a pseudo-terminal on the server. Once the commands are run on the server, the output is relayed back to the client.

The Record Label Admin Panel was intended for management of a record shop, providing administrator tools for a database of records. A responsive User Interface, designed with bootstrap, allows the system admin basic C.R.U.D. functions, such as creating new records, searching through the record catalogue, updating records, and deleting records. This project also features a Django webserver and relies on Django HTML templates for listing record data.

This functions similarly to a VPN by routing all URL to a remote server. The HTML returned to the client is displayed inside a WebView object from the JavaFX library.
